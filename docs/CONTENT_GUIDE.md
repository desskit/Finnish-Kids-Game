# Content guide — growing the learning path

This is the practical guide for adding content to Kielipesä. The app is organized
as a **journey path** of chapters → skill nodes (see `src/game/path.tsx`), built
around *usable Finnish* (things you can say). Vocabulary is the material that
flows through the communicative skills.

## The one golden rule

**Never generate or inflect Finnish in code.** Every Finnish form shown to a
child is *looked up* from human-sourced, tagged data (Wiktionary via kaikki.org,
CC BY-SA 4.0). If a word form you need isn't in the data, the word can't be used
until the form is added to the source. This is what keeps the Finnish correct
(consonant gradation, agreement, rection, …).

So adding content is usually two steps: **(1)** make sure the words exist in the
sourced data, then **(2)** wire them into a construction / skill / sentence.

---

## 1. Add vocabulary (words)

Words live in `src/content/data/*.sourced.json`, generated by the curation
script from the vendored dataset.

1. Edit the curation lists (`ANIMALS`, `FOOD`, `FAMILY`, `NUMBERS`, `ADJECTIVES`,
   `VERBS`) in `scripts/build-kids-data.mjs` — add the word's lemma + English gloss
   (+ an emoji until art lands).
2. Regenerate:
   ```bash
   node scripts/build-kids-data.mjs data/finnish-inflection-drill
   ```
   This copies the verified inflection table + tags into the `*.sourced.json`.
3. If the word isn't in the upstream dataset, it must be added there first (see
   `data/finnish-inflection-drill/README.md`). Don't hand-type inflections.
4. `npm run test:unit` — the content-integrity test (`src/content/content.test.ts`)
   checks ids are unique and every construction can be filled.

New words automatically join the mixed noun pool the path's skills draw from, and
the spaced-repetition Review.

## 2. Add a carrier phrase (single-slot construction)

A `Construction` (`src/content/constructions.ts`) is a phrase with **one**
inflected slot, e.g. `Tämä on ___`. Add an entry with the fixed words + the
`case` + `number` the slot must take (the form is looked up by those tags):

```ts
{ id: 'i-want', before: 'Haluan', punct: '.', en: 'I want a ___.',
  tier: 3, case: 'partitive', number: 'singular' }
```

Then expose it as a skill (step 4). Make sure the chosen case exists for the words
you'll use (the integrity test will catch gaps for the noun topics).

See `docs/FINNISH_GRAMMAR.md` for which case/number a construction needs for
possession, negation, partitive-government verbs and counting — it's the
authoring contract behind the `case`/`number` tags above.

## 3. Add a skill node / chapter (the path)

The path is data in `src/game/path.tsx`. A **skill node** picks an existing game
(`activity`) + a content scope:

```ts
{ id: 'i-want', titleFi: 'Haluan…', titleEn: 'I want…', icon: '🙋',
  activity: 'build', content: { constructionIds: ['i-want'] }, exampleFi: 'Haluan omenaa.' }
```

- `activity`: `listen | build | count | match | conjugate | order | spell | sentence | review`
- `content.pool`: which words — `'nouns'` (all noun topics mixed, default),
  `'animals' | 'food' | 'family' | 'numbers'`.
- `content.constructionIds`: for `build`/`order`, which carrier phrases to drill
  (omit = all).

Add it to a chapter's `skills`, or add a whole new `Chapter` (give it a `titleFi/En`,
an `accent` color, an `icon`). No router or component changes are needed —
progress, adaptive difficulty, badges and the dashboard all pick it up by id.

## 4. Add a multi-slot sentence (the "Full sentences" chapter)

This is the advanced chapter — **wired but intentionally empty** today. Authoring
templates in `src/content/sentences.ts` makes playable nodes appear automatically
(`buildSentenceRound` turns each into a Word-Order round; see `src/game/round.ts`).

A `SentenceConstruction` is an ordered list of `tokens` (a fixed word or a `slot`)
plus the `slots` themselves. Each slot pulls a form from the data; **agreement is
resolved by copying tags** (never by rule):

```ts
// "Minä annan koiralle luun."  (subject + verb + recipient + object)
{
  id: 'give-recipient-object',
  en: 'I give the dog a bone.',
  tier: 4,
  punct: '.',
  tokens: [{ slot: 'subject' }, { slot: 'verb' }, { slot: 'recipient' }, { slot: 'object' }],
  slots: [
    { id: 'subject',   role: 'pronoun', fixedId: '1sg' },
    { id: 'verb',      role: 'verb', verbSlotForm: 'conjugated',
      tense: 'present', polarity: 'positive', agreesWith: 'subject', fixedId: 'antaa' },
    { id: 'recipient', role: 'noun', case: 'allative', number: 'singular', pool: 'nouns' },
    { id: 'object',    role: 'noun', case: 'genitive', number: 'singular', pool: 'nouns' },
  ],
}
```

Patterns this supports:
- **Two objects / recipient + object** — two `noun` slots in different cases.
- **Adjective + noun object** — an `adjective` slot with `agreesWith` its noun slot
  (it copies the noun's case + number): `Näen ison koiran`.
- **Verb chains (multiple verbs)** — one `verb` slot `conjugated`, another with
  `verbSlotForm: 'infinitive'`: `Haluan syödä omenan`.

Slot fields: `role` (`noun|adjective|verb|pronoun|number`), `case` + `number`
(nouns/adjectives), `tense` + `polarity` + `verbSlotForm` (verbs), `agreesWith`
(another slot id), `pool` (draw a random word) or `fixedId` (pin one word/pronoun).
Pronoun ids are `1sg,2sg,3sg,1pl,2pl,3pl`.

Tips: pin words with `fixedId` while drafting so you can read the exact sentence;
the resolver returns nothing for a slot whose form is missing in the data (so make
sure the needed case/conjugation exists). The shape is **v1** — expect it to gain
fields (optional slots, alternate word orders) as real templates arrive.

## 5. Art integration (later)

The path is built to drop art in as a **data/CSS change, not a rewrite**:

- **Node / chapter art:** set `art` on a `SkillNode`, or `bannerArt` / `bgArt` on a
  `Chapter`. The emoji `icon` stays as the fallback. Reference assets via
  `import.meta.env.BASE_URL` (e.g. `art/path/this-is.webp`) so they work on
  Netlify, a Pages subpath, and a Capacitor `file://` build. Use WebP, ~512²,
  transparent — matching the Phase 1 art conventions in `ROADMAP.md`.
- **Per-chapter color:** change a chapter's `accent`; it themes the band, node
  rings and the "next" highlight.
- **Re-fitting the layout to an illustrated map:** nodes auto-arrange into the
  serpentine now. Each node also accepts an optional `pos: { x, y }` (percent) and
  `side` hint, so once you have a painted map you can place nodes at exact
  coordinates over it by editing data only (the renderer prefers `pos` when set).

## Where your help is most valuable (content wishlist)

The path is buildable from today's content, but these unlock new chapters and make
the Finnish more *usable*. For each: provide Finnish + English; confirm the words
exist in (or can be added to) the inflection dataset.

1. **Greetings & classroom phrases** — `Moi! Mitä kuuluu? Kiitos. Anteeksi.` Enables
   a future "Conversations" chapter (needs a small dialogue content type — ask and
   I'll spec it).
2. **More everyday verbs** — richer conjugation + the verb-chain sentences above.
3. **Question words** — `kuka / mikä / missä / montako` for question skills.
4. **Colors** (as adjectives) — feed the agreement game and descriptive sentences.
5. **More high-frequency nouns** across home / outdoors / body / clothes.
6. **Multi-slot sentence templates** (§4) — the manual-but-high-impact work.
